# Hosting and Data Sources

이 문서는 `mcp-darksky`를 직접 배포하거나 외부 데이터 소스를 함께 사용할 때 참고할 운영 메모입니다. 법률 자문을 대신하지 않으며, 실제 사용 전에는 각 제공자의 최신 정책과 약관을 직접 확인하는 편이 안전합니다.

## Public endpoint policy

공개 엔드포인트가 제공되더라도 다음 전제로 이해하는 것을 권장합니다.

- 개인 평가와 가벼운 대화형 테스트용
- rate limit, 변경, 중단 가능
- 가용성이나 지원 보장 없음

다음 용도는 직접 호스팅을 권장합니다.

- 반복 호출
- 자동화
- 팀 공유
- 제품 연동
- 공개 서비스에 포함되는 사용

## Self-hosting

직접 호스팅 시에는 본인 인프라와 본인 API 키를 사용하세요.

운영 전 점검 항목:

1. `PUBLIC_BASE_URL`, `KAKAO_REST_API_KEY` 등 환경 변수를 직접 설정
2. 광해 추정에 필요한 runtime artifact와 로컬 데이터 준비
3. 테스트와 헬스 체크 확인
4. 공개 사용량과 비용을 본인 계정 기준으로 관리

## Third-party services and data

이 프로젝트는 다음 외부 제공자를 함께 사용할 수 있습니다.

- Open-Meteo
- Kakao Local API
- NASA Black Marble 계열 데이터

README와 문서에서 외부 제공자를 언급하는 이유는 데이터 흐름을 설명하기 위해서입니다. 각 제공자의 약관, 허용 사용 범위, 할당량, 인증 방식, 출처 표기 요구사항은 사용자가 직접 확인하고 준수해야 합니다.

문서에서는 다음 원칙을 따릅니다.

- 특정 제공자의 정책을 이 저장소가 대신 해석하지 않음
- 외부 서비스 사용 가능 여부를 법률적으로 단정하지 않음
- 실제 서비스 운영 시에는 사용자가 본인 키와 본인 인프라로 책임 범위를 분리함

## Attribution and endorsement

외부 데이터와 서비스를 출처로 언급하는 것은 설명 목적입니다. 이는 해당 제공자가 `mcp-darksky`를 승인, 후원, 보증한다는 뜻이 아닙니다.

출처 표기가 필요하거나 권장되는 경우에는 사실 그대로 표기하되, 제3자 보증으로 읽히는 표현은 피하는 것이 좋습니다.

## Practical documentation pattern

프로젝트 문서에서는 아래 수준으로 안내하는 것이 안전합니다.

- 공개 주소는 개인 테스트용
- 넓은 활용은 self-host 권장
- 외부 데이터와 API는 각자 정책 확인 필요
- 외부 제공자 이름은 설명용이며 보증 의미 없음

이 수준을 넘는 세부 정책 판단은 배포자나 사용자가 공식 문서를 기준으로 별도로 검토하는 편이 적절합니다.
